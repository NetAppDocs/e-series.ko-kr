.단계
. 이 절차를 위해 가동 중지 유지보수 윈도우를 예약하십시오. 이 절차를 성공적으로 완료할 때까지 스토리지 시스템의 데이터에 액세스할 수 없습니다. 두 컨트롤러 모두 전원이 켜진 상태에서 동일한 I/O 모듈 구성을 가져야 하므로 I/O 모듈을 업그레이드할 때는 전원을 꺼야 합니다.
. 다음 사항을 확인하십시오.
+
** 업그레이드하려는 새로운 I/O 모듈.
+
업그레이드하려는 I/O 모듈이 스토리지 시스템에서 지원되는지, 각 컨트롤러에 동일한 유형과 개수의 I/O 모듈이 있는지, 그리고 슬롯 우선순위(새 I/O 모듈은 각 컨트롤러의 동일한 슬롯에 설치해야 함)를 이해하고 있는지 확인하십시오.

+

CAUTION: *데이터 액세스 손실 가능성* -- 호환되지 않거나 일치하지 않는 I/O 모듈이 있는 경우 전원을 공급하면 컨트롤러가 잠깁니다.

+
필요한 경우 스토리지 시스템에 지원되는 I/O 모듈 및 슬롯 우선 순위를 다음에서 확인할 수 있습니다 https://hwu.netapp.com["NetApp Hardware Universe를 참조하십시오"^].

** 새 I/O 모듈을 네트워크에 연결하는 케이블.
** ESD 손목 밴드를 착용했거나 다른 정전기 방지 조치를 취했습니다.
** 정전기가 없는 평평한 작업 표면
** 컨트롤러용 SANtricity System Manager에 액세스할 수 있는 브라우저가 있는 관리 스테이션.
+
System Manager 인터페이스를 열려면 브라우저에서 대상 컨트롤러의 도메인 이름 또는 IP 주소를 가리킵니다.



. SANtricity System Manager를 사용하여 스토리지 시스템의 구성 데이터베이스를 백업하십시오.
+
컨트롤러를 제거할 때 문제가 발생하면 저장된 파일을 사용하여 구성을 복원할 수 있습니다. 시스템은 장애가 발생한 컨트롤러의 볼륨 그룹 및 디스크 풀에 대한 모든 데이터를 포함하는 RAID 구성 데이터베이스의 현재 상태를 저장합니다.

+
.. 지원 * > * 지원 센터 * > * 진단 * 을 선택합니다.
.. 구성 데이터 수집 * 을 선택합니다.
.. *Collect*를 선택합니다.
+
파일은 브라우저의 다운로드 폴더에 * configurationData - <arrayName> - <DateTime>.7z * 라는 이름으로 저장됩니다.
