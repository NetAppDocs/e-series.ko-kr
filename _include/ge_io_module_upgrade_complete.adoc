.단계
. 최신 버전의 SANtricity OS가 설치되어 있는지 확인하십시오.
+
.. *지원* > *업그레이드 센터*를 선택합니다.
.. 필요한 경우 최신 버전을 설치합니다.


. 모든 볼륨이 소유 컨트롤러로 반환되었는지 확인합니다.
+
.. * Storage * > * Volumes * 를 선택합니다.
.. *All Volumes* 페이지에서 볼륨이 해당 볼륨을 소유하는 컨트롤러에 배포되었는지 확인하십시오.
.. 볼륨 소유자를 보려면 *More* > *Change volume ownership*을 선택하세요.


. 다음 단계는 볼륨이 소유 컨트롤러로 반환되는지 여부에 따라 달라집니다.
+
[cols="1,2"]
|===
| 만약... | 그런 다음... 


 a| 
볼륨은 모두 해당 소유 컨트롤러가 소유합니다
 a| 
다음 단계로 이동합니다.



 a| 
어떤 볼륨도 소유 컨트롤러로 반환되지 않습니다
 a| 
*Storage* > *Volumes* > *More*를 선택하고 드롭다운 메뉴에서 *Redistribute volumes*를 선택하여 볼륨을 소유 컨트롤러로 되돌립니다.



 a| 
자동 배포 또는 수동 배포 후 일부 볼륨만 소유 컨트롤러로 반환됩니다.
 a| 
호스트 연결 문제에 대해 Recovery Guru를 확인하십시오.

Recovery Guru가 없거나 Recovery Guru 단계를 수행한 후에도 볼륨이 소유 컨트롤러로 반환되지 않으면 https://mysupport.netapp.com/site/global/dashboard["NetApp 지원"]이 절차를 계속하기 전에 문의하십시오.

|===
. 스토리지 시스템에 대한 지원 데이터 수집:
+
.. 지원 * > * 지원 센터 * > * 진단 * 을 선택합니다.
.. 지원 데이터 수집 * 을 선택합니다.
.. *Collect*를 선택합니다.
+
파일은 브라우저의 다운로드 폴더에 * support-data.7z * 라는 이름으로 저장됩니다.



. 호스트 I/O 모듈 프로토콜을 확인하고 변경해야 하는 경우 xref:io-module-change-protocol.html[EF50 또는 EF80의 호스트 I/O 모듈 프로토콜을 변경합니다] 절차를 사용하십시오.
