.단계
. 교체용 컨트롤러를 포장에서 꺼내어 정전기가 없는 평평한 표면에 놓으십시오.
+
장애가 발생한 컨트롤러를 반환할 때 사용할 포장재를 보관하십시오.

. 교체용 컨트롤러 측면에 있는 MAC 주소를 기록해 두십시오.
+
MAC 주소는 컨트롤러 교체 절차의 후반부에 사용됩니다.

. 교체용 컨트롤러의 부품 번호가 SANtricity System Manager에 등록된 컨트롤러의 교체 부품 번호와 동일한지 확인하십시오.
+

CAUTION: *데이터 액세스 손실 가능성*: 두 부품 번호가 동일하지 않으면 이 절차를 시도하지 마십시오. 컨트롤러가 일치하지 않으면 교체 컨트롤러를 온라인 상태로 전환할 때 잠금 상태가 됩니다.

+
.. 교체용 컨트롤러 측면에 있는 부품 번호를 기록해 두십시오.
.. Recovery Guru의 세부 정보 영역에 표시된 손상된 컨트롤러의 교체 부품 번호를 찾으십시오.
+
이 번호를 수동으로 찾아야 하는 경우 다음 단계를 따르십시오.

+
... * Hardware * > * Controllers and components * 를 선택하십시오.
... 손상된 컨트롤러(컨트롤러 A 또는 컨트롤러 B)에 해당하는 컨트롤러 그래픽을 선택하여 상황별 메뉴를 표시하십시오.
... 설정 보기 * 를 선택합니다.
... 기본 * 탭에서 컨트롤러의 * 교체 부품 번호 * 를 기록해 둡니다.


.. 두 부품 번호를 비교하여 동일한지 확인하십시오.
.. 다음 단계는 부품 번호가 동일한지 여부에 따라 달라집니다.
+

. SANtricity System Manager를 사용하여 스토리지 시스템의 구성 데이터베이스를 백업하십시오.
+
컨트롤러를 제거할 때 문제가 발생하면 저장된 파일을 사용하여 구성을 복원할 수 있습니다. 시스템은 컨트롤러의 볼륨 그룹 및 디스크 풀에 대한 모든 데이터를 포함하는 RAID 구성 데이터베이스의 현재 상태를 저장합니다.

+
.. 지원 > 지원 센터 > 진단 * 을 선택합니다.
.. 구성 데이터 수집 * 을 선택합니다.
.. *Collect*를 선택합니다.
+
파일은 브라우저의 다운로드 폴더에 * configurationData - <arrayName> - <DateTime>.7z * 라는 이름으로 저장됩니다.
