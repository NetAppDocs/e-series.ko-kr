= 컨트롤러 교체 워크플로 - EF50 및 EF80
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
스토리지 시스템 요구 사항을 충족하고, 시스템을 준비하고, 손상된 컨트롤러를 오프라인으로 전환하고, 컨트롤러를 교체하고, 새 컨트롤러를 온라인으로 전환하고, 교체 프로세스를 완료하여 EF50 또는 EF80 스토리지 시스템에서 손상된 컨트롤러를 교체할 수 있습니다.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["1개"]link:controller-replace-requirements.html["컨트롤러 교체 요구 사항을 검토하십시오"]
[role="quick-margin-para"]
스토리지 시스템에 장애가 발생한 컨트롤러가 있는지 확인하고, 스토리지 시스템의 상태를 확인하고, 절차에 필요한 도구를 확보하고, SANtricity System Manager에 액세스할 수 있는 관리 스테이션과 브라우저를 준비하십시오.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["2개"]link:controller-replace-prepare.html["컨트롤러 교체 준비"]
[role="quick-margin-para"]
교체용 컨트롤러를 개봉하고 MAC 주소를 기록해 두고 부품 번호가 올바른지 확인한 후 스토리지 시스템의 구성 데이터베이스를 백업하십시오.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["세 가지"]link:controller-replace-offline-controller.html["손상된 컨트롤러를 오프라인으로 전환합니다"]
[role="quick-margin-para"]
손상된 컨트롤러를 교체할 때 데이터 손실을 방지하고 시스템 안정성을 확보하려면 손상된 컨트롤러를 오프라인으로 전환하십시오.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["네"]link:controller-replace-move-hardware.html["손상된 컨트롤러를 교체합니다"]
[role="quick-margin-para"]
캐시 메모리의 데이터가 드라이브에 기록되었는지 확인하고, 손상된 컨트롤러에서 모든 케이블을 분리하고, 컨트롤러를 제거하고, 구성 요소를 교체용 컨트롤러로 옮기고, 교체용 컨트롤러를 설치한 후 모든 케이블을 다시 연결하십시오.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["다섯"]link:controller-replace-online-controller.html["교체 컨트롤러를 온라인으로 전환합니다"]
[role="quick-margin-para"]
교체 컨트롤러의 IP 주소를 확인하고, 컨트롤러를 온라인 상태로 전환한 후, 스토리지 시스템 상태가 최적인지 확인하십시오.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["6개"]link:controller-replace-complete-replacement.html["컨트롤러 교체 완료"]
[role="quick-margin-para"]
최신 버전의 SANtricity OS가 설치되어 있는지 확인하고, 볼륨이 해당 컨트롤러로 반환되었는지 확인하고, 시스템 지원 데이터를 수집한 후, 고장난 부품을 NetApp으로 반환하십시오.
